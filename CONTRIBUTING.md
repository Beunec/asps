# Contributing to ASPS™

Thank you for your interest in contributing to the Agentic-System-Prompt-as-a-Skill™ framework.

## How to Contribute

### Reporting Issues

- Use [GitHub Issues](https://github.com/Beunec/asps/issues) for bugs and feature requests.
- Include the package name (`beunec-asps` or `@beunec/asps`), version, and steps to reproduce.

### Suggesting Features

- Open a GitHub Issue with the `enhancement` label.
- Describe the use case, expected behavior, and how it aligns with the ASPS™ three-layer framework (ASD™, ASR™, ANS™).

### Submitting Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Ensure all tests pass
5. Submit a pull request against `main`

### Development Setup

```bash
# Clone
git clone https://github.com/Beunec/asps.git
cd asps
```

### Coding Standards

- **TypeScript:** Follow the existing `tsconfig.json` strict mode conventions. Zero dependencies.
- **Python:** Follow PEP 8. Type hints required. Zero dependencies.
- Keep both packages feature-equivalent — changes to one should be mirrored in the other.

### What We Accept

- Bug fixes with tests
- New skill templates with complete heuristic chains
- New guardrail/checkpoint presets
- Documentation improvements
- Performance optimizations

### What We Don't Accept

- Changes that introduce external runtime dependencies
- Breaking changes to the public API without prior discussion
- Content that violates the [Code of Conduct](CODE_OF_CONDUCT.md)

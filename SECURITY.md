# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | ✅         |

## Reporting a Vulnerability

If you discover a security vulnerability in ASPS™, **please do not open a public issue.**

Instead, report it privately via one of the following:

- **Email:** [security@beunec.com](mailto:contact@beunec.com) or [support@beunec.com](mailto:support@beunec.com)
- **Web:** [https://beunec.co](https://beunec.co)

Please include:

1. A description of the vulnerability
2. Steps to reproduce
3. Affected package(s) (`beunec-asps`, `@beunec/asps`, or both)
4. Any potential impact assessment

## Response Timeline

- **Acknowledgement:** within 48 hours
- **Initial assessment:** within 5 business days
- **Patch release:** as soon as a fix is verified

## Scope

This policy covers the ASPS™ framework code and its published packages:

- [beunec-asps](https://pypi.org/project/beunec-asps/) (PyPI)
- [@beunec/asps](https://www.npmjs.com/package/@beunec/asps) (npm)

Third-party dependencies are outside our direct scope, but we will coordinate upstream if needed.
